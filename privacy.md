# Keeping 개인정보 처리방침
{: .no_toc}

**시행일: 2026년 4월 20일 · 최종 수정일: 2026년 4월 20일**

<div class="summary" markdown="1">

### 핵심 요약
{: .no_toc}

- **회원 가입·로그인 없음**: 모든 저장 콘텐츠는 이용자의 기기 내부에만 보관됩니다.
- **서버 미저장**: AI 분석 요청 시 URL·제목·메모·썸네일 URL이 일시 전송되며, 분석 직후 파기됩니다.
- **수집하지 않는 것**: 익명 식별자(UUID), 기기 정보, 크래시 로그, 푸시 토큰, 광고 식별자
- **위탁 처리**: Supabase(AI 분석 중계), Google Gemini(AI 분석) — 모두 미국, 처리 즉시 파기
- **권리**: 앱 내 "계정 및 모든 데이터 삭제"로 즉시 삭제 가능
- **문의**: keeping.app.support@gmail.com

Keeping(이하 "앱" 또는 "서비스")을 운영하는 개발자(이하 "운영자")는 이용자의 개인정보를 소중히 여기며, 「개인정보 보호법」 및 「정보통신망 이용촉진 및 정보보호 등에 관한 법률」 등 관련 법령을 준수합니다. 본 처리방침은 운영자가 이용자의 개인정보를 어떻게 수집·이용·보관·파기하는지 안내합니다.

## 1. 수집하는 개인정보 항목 및 수집 방법

### 가. 이용자가 직접 입력·제공하는 정보 (기기 내 저장)
- 저장 콘텐츠: URL, 페이지 제목·설명, 썸네일 이미지 URL, 사용자 메모, 태그, 카테고리 이름
- 설정 정보: 알림 수신 여부, 카테고리 정렬, 위젯 상태 등 앱 내 환경 설정

위 정보는 **이용자의 기기 저장소(SharedPreferences)** 에만 기록되며, 앱을 삭제하면 함께 삭제됩니다.

### 나. AI 분석 요청 시 일시적 전송 항목 (서버 미저장)

이용자가 저장 액션(URL 복사·공유) 또는 편집을 수행할 때, AI 분석을 위해 아래 항목이 **Supabase Edge Function을 경유하여 Google Gemini API** 로 전송됩니다.

- 저장한 URL
- 해당 URL의 페이지 제목·설명 (HTML `<meta>`/OpenGraph에서 추출)
- 이용자가 입력한 메모 (해당 항목의 AI 재분석을 명시적으로 요청한 경우에 한함)
- 썸네일 이미지 URL (Gemini Vision 분석이 가능한 이미지 중심 사이트에 한함)

위 정보는 AI 분석 응답 생성 직후 서버 처리 메모리에서 파기되며, **Supabase 및 Google은 해당 입력을 자사 AI 모델 학습에 사용하지 않습니다** (Gemini API 기본 정책).

### 다. 수집하지 않는 항목
- 이름·이메일·전화번호·주소 등 일반 개인정보
- 익명 식별자(UUID), 광고 식별자(IDFA/AAID)
- 기기 모델·OS 버전·앱 버전 등 기기 정보
- 위치 정보, 연락처, 사진첩, 캘린더
- 크래시 로그, 사용 분석(Analytics), 푸시 토큰

### 라. 수집 방법
- 이용자가 앱 내에서 직접 콘텐츠를 저장하거나 입력하는 방식

## 2. 개인정보의 처리 목적

운영자는 수집한 정보를 다음 목적으로만 이용합니다.

1. **서비스 제공**: 이용자가 저장한 콘텐츠의 로컬 저장·조회·검색·재발견(리서페이스)
2. **AI 분석**: 저장 콘텐츠의 제목 정제, 요약, 태그·카테고리 자동 분류

---

## 3. 개인정보의 보유 및 이용 기간

| 항목 | 보유 위치 | 보유 기간 |
|---|---|---|
| 저장 콘텐츠(URL, 텍스트, 설정) | **이용자 기기 내부** | 이용자가 앱을 삭제하거나 앱 내 "계정 및 모든 데이터 삭제" 메뉴를 선택할 때까지 |
| AI 분석 요청 데이터 | Supabase Edge Function / Google Gemini | **미저장**. 응답 생성 후 즉시 파기 |

---

## 4. 개인정보의 제3자 제공

운영자는 이용자의 개인정보를 제3자에게 제공하지 않습니다. 다만, 법령의 규정에 의거하거나 수사기관의 적법한 요청이 있는 경우는 예외로 합니다.

---

## 5. 개인정보 처리의 위탁 및 국외 이전

AI 분석 기능 제공을 위해 아래와 같이 개인정보 처리를 위탁하며, 해당 업무 수행 과정에서 국외 이전이 발생합니다.

| 수탁 업체 | 소재지 | 위탁 업무 | 이전 항목 | 이전 시점 | 보유 기간 |
|---|---|---|---|---|---|
| Supabase Inc. | 미국 | AI 분석 요청 중계(Edge Function) | URL, 페이지 제목·설명, 메모, 썸네일 URL | AI 분석 요청 시 네트워크 전송 | 처리 즉시 파기 (서버 미저장) |
| Google LLC | 미국 | AI 분석(Gemini API — 제목·요약·태그·카테고리 생성) | 상동 | AI 분석 요청 시 네트워크 전송 | 처리 즉시 파기 (Gemini API 기본 정책, 모델 학습에 사용되지 않음) |

이전 방법: HTTPS(TLS) 기반 네트워크 전송

이용자는 AI 분석 기능을 사용하지 않기 위해 앱을 이용하지 않을 수 있으나, 앱의 핵심 기능 특성상 해당 기능을 개별 거부할 수 있는 설정은 현재 제공되지 않습니다. 향후 버전에서 선택적 비활성화 옵션을 추가할 수 있습니다.

---

## 6. 이용자의 권리와 행사 방법

이용자는 언제든지 다음 권리를 행사할 수 있습니다.

1. **본인 데이터 열람·삭제**: 저장된 모든 콘텐츠는 앱 내에서 직접 조회하고 개별·일괄 삭제할 수 있습니다.
2. **전체 데이터 초기화**: 앱 내 `설정 → 계정 → 계정 및 모든 데이터 삭제` 메뉴에서 모든 로컬 데이터를 즉시 영구 삭제할 수 있습니다. 삭제 후 앱은 자동으로 재시작됩니다.
3. **앱 삭제를 통한 탈퇴**: 기기에서 앱을 삭제하면 로컬 데이터도 함께 제거됩니다.
4. **기타 문의**: 개인정보 보호책임자 이메일로 요청 시 지체 없이(최대 10일 이내) 처리합니다.

AI 분석 요청 데이터는 서버에 저장되지 않으므로 별도의 열람·삭제 절차가 필요하지 않습니다.

---

## 7. 개인정보의 파기 절차 및 방법

- 파기 시점: 이용자의 삭제 요청, 앱 삭제, 또는 "계정 및 모든 데이터 삭제" 선택 시
- 파기 방법: 전자적 파일 형태의 정보는 운영체제의 영속 저장소 API를 통해 복구 불가능하게 제거

---

## 8. 개인정보의 안전성 확보 조치

- **전송 구간 암호화**: 앱과 서버 간 통신 전 구간 HTTPS(TLS) 적용
- **최소 수집 원칙**: 서비스 제공에 꼭 필요한 정보만, 그리고 가능한 경우 기기 내부에만 저장
- **서버 미저장 원칙**: AI 분석 요청 데이터는 서버에 저장하지 않고 응답 생성 즉시 파기
- **API 키 보호**: 외부 AI 서비스 API 키는 서버 환경변수에만 존재하며 클라이언트에 노출되지 않음

## 9. 앱 접근 권한 안내

Keeping은 서비스 제공을 위해 아래와 같이 기기 권한에 접근합니다.

### 필수 접근 권한
- **인터넷(네트워크)**: AI 분석 서버 통신 및 저장한 URL의 썸네일·메타데이터 가져오기

### 선택 접근 권한 (거부해도 서비스 이용 가능)
- **알림**: 저장 콘텐츠 재발견 및 리마인드 알림 (기기 내부에서 생성되는 로컬 알림만 사용)

이용자는 기기 설정에서 언제든지 권한을 재설정할 수 있습니다. 카메라·사진첩·위치·연락처·마이크 등 민감 권한은 요청하지 않습니다.
